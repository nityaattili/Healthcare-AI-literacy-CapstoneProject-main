# AI Literacy – Clinical Literature Analysis

A web-based application for analyzing healthcare literature using NLP: topic modeling, keyword trends, author/journal statistics, and co-citation networks. Built per the **AI Literacy Proposal** (clinical literature analysis using NLP and an interactive web app).

## Features

- **Data**: Fetch from PubMed API or load CSV/JSON; includes demo dataset (500+ papers).
- **NLP**: Text preprocessing (spaCy/NLTK), LDA topic modeling, keyword extraction.
- **Analytics**: Author and journal statistics, co-citation network analysis.
- **Web app**: Streamlit UI to explore topics, keyword trends, author/journal stats, and co-citation maps.

## Setup

### 1. Virtual environment and dependencies

```bash
python3 -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

### 2. Download spaCy model (for preprocessing)

```bash
python -m spacy download en_core_web_sm
```

### 3. Download NLTK data (first run)

```bash
python -c "import nltk; nltk.download('stopwords'); nltk.download('wordnet'); nltk.download('punkt')"
```

## Run End-to-End

### Option A: One-time pipeline then app

**Using synthetic demo data (no API, quick test):**
```bash
# 1a. Generate a small synthetic dataset (no PubMed needed)
python scripts/generate_demo_csv.py

# 2. Run full pipeline (preprocess → topics → keywords → stats → co-citation)
python scripts/run_pipeline.py

# 3. Launch web app
streamlit run src/app/main.py
```

**Using real PubMed data:**
```bash
# 1b. Download demo data from PubMed (requires network)
python scripts/download_demo_data.py

# 2. Run full pipeline
python scripts/run_pipeline.py

# 3. Launch web app
streamlit run src/app/main.py
```

### Option B: App only (uses existing output)

If `output/` already has results from a previous pipeline run:

```bash
streamlit run src/app/main.py
```

### Option C: Docker

```bash
docker build -t ai-literacy-app .
docker run -p 8501:8501 ai-literacy-app
```

Then open http://localhost:8501.

## Data Format

- **Input**: CSV or JSON with columns such as `title`, `abstract`, `authors`, `journal`, `year`, `keywords`, `pmid` (or `id`). PubMed fetcher produces this format.
- **Demo**: `data/demo/papers.csv` (generated by `download_demo_data.py`).

## Tech Stack

- **Python**: pandas, requests, Biopython (PubMed)
- **NLP**: spaCy, NLTK, gensim (LDA), scikit-learn
- **Analytics**: pandas, NetworkX (co-citation)
- **Frontend**: Streamlit, Plotly, Matplotlib
